模块零题目与答案代码

先把题目要做什么、以及这段代码到底写了什么放在最前面。建议先扫一遍题意，再看右侧这份带中文注释的代码，后面的可视化和练习都会严格对应它来展开。

题目要求
给定两个字符串 s 和 t， 统计在 s 中有多少个不同的子序列，能够恰好等于 t。
  • 子序列的意思是：可以删除一些字符，也可以一个都不删，但不能打乱原来的相对顺序。
  • “不同”通常体现在选择位置不同。哪怕拼出来的字符一样，只要选的位置不同，也算不同方案。
  • 这道题的关键不是暴力枚举，而是把“前缀能拼出前缀”的问题拆成很多小状态。

这份答案代码在做什么
  • 先定义 dp[i][j] 的含义。
  • 先处理最基础的情况：目标串为空时答案恒为 1。
  • 然后按代码顺序双层循环填表。
  • 如果当前字符相等，就把“使用它”和“不使用它”两种方案相加；不相等就只能跳过它。
带中文注释的参考答案
展示用代码，和下方教学模块完全对应
/**
 * 115. 不同的子序列
 * 动态规划
 *
 * dp[i][j] 表示：
 * 在 s 的前 i 个字符中，
 * 组成 t 的前 j 个字符的不同子序列个数
 */
var numDistinct = function(s, t) {
  const m = s.length;
  const n = t.length;

  // 创建 (m + 1) x (n + 1) 的 dp 表
  // 多出来的一行一列，用来表示“空前缀”
  const dp = Array.from({ length: m + 1 }, () => Array(n + 1).fill(0));

  // 初始化第 0 列：
  // t 的前 0 个字符就是空串
  // 不管 s 取前多少个字符，组成空串都只有 1 种方法：什么都不选
  for (let i = 0; i <= m; i++) {
    dp[i][0] = 1;
  }

  // dp[0][j] 默认保持 0（j > 0）
  // 因为空字符串 s 不可能组成非空字符串 t

  // 开始填表
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      // 当前比较的是 s[i - 1] 和 t[j - 1]
      if (s[i - 1] !== t[j - 1]) {
        // 字符不相等：
        // 当前 s 字符不能拿来匹配 t 的当前字符
        // 只能跳过它，继承上方结果
        dp[i][j] = dp[i - 1][j];
      } else {
        // 字符相等时有两种选择：
        //
        // 1. 使用 s[i - 1] 去匹配 t[j - 1]
        //    那前面要在 s 的前 i - 1 个字符里组成 t 的前 j - 1 个字符
        //    对应 dp[i - 1][j - 1]
        //
        // 2. 不使用 s[i - 1]
        //    继续在 s 的前 i - 1 个字符里组成 t 的前 j 个字符
        //    对应 dp[i - 1][j]
        dp[i][j] = dp[i - 1][j - 1] + dp[i - 1][j];
      }
    }
  }

  // 右下角就是最终答案
  return dp[m][n];
};
