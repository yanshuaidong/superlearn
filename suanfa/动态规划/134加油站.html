134. 加油站：把题目过程“转起来”

这题最容易绕的地方只有一句话：每到一个站，先加上该站的油 `gas[i]`，再开去下一个站，路上消耗 `cost[i]`。 下面把示例 1 和示例 2 都做成了动态过程图，你可以一步一步看油箱怎么变化。

站点含义 每个站点都标了 `gas` 和 `cost`。`gas` 是你在这个站能拿到多少油，`cost` 是从这个站开到下一个站要花多少油。
看油箱公式 每一步都遵循：`到站前油量 + 当前站加油 - 开往下一站消耗 = 到达下一站后的油量`。
成功与失败 只要某一步算出来油量变成负数，就说明这条出发路线走不通；如果能回到起点且油量不为负，就说明能绕一圈。

代码怎么跑：左边看代码，右边看“人话版”执行过程

这里用的是你写的示例 1 推演。左边是标准代码，右边把循环里每一轮发生的事情翻译成自然语言，这样更容易理解为什么最后 `start` 会停在下标 `3`。

左侧：算法代码
/**
 * @param {number[]} gas
 * @param {number[]} cost
 * @return {number}
 */
var canCompleteCircuit = function (gas, cost) {
  let total = 0;
  let tank = 0;
  let start = 0;

  for (let i = 0; i < gas.length; i += 1) {
    const gain = gas[i] - cost[i];
    total += gain;
    tank += gain;

    if (tank < 0) {
      start = i + 1;
      tank = 0;
    }
  }

  if (total < 0) {
    return -1;
  }

  return start;
};
右侧：示例 1 的自然语言执行过程
先记住 3 个变量分别在干嘛

`total` 表示把每个站点的净收益一个一个加起来以后，整个环路总共还剩多少油。

`tank` 表示从当前候选起点出发，走到当前下标时，当前这条路线里还剩多少油。

`start` 表示当前这一轮认为最可能成为起点的下标。

开始前

输入是：`gas = [1,2,3,4,5]`，`cost = [3,4,5,1,2]`。

开始时，`total` 是 0，`tank` 是 0，`start` 是 0。

也就是说，一开始先假设可以从 0 号站出发。

第 1 轮：i = 0

当前下标是 0。

先算当前站点的净收益。0 号站可以加 1 升油，从 0 号站开到 1 号站要花 3 升油，所以这个站点的净收益是负 2。

把这个结果累加到全局总油量里。原来的 `total` 是 0，加上负 2 以后，`total` 变成负 2。

再把这个结果累加到当前路径剩余油量里。原来的 `tank` 是 0，加上负 2 以后，`tank` 变成负 2。

现在 `tank` 已经小于 0，说明如果从 0 号站出发，走到当前这里就已经走不下去了。

所以 0 号站不能作为起点。

把下一个站，也就是 1 号站，设为新的候选起点，所以 `start` 变成 1。

同时把 `tank` 重置为 0，表示从新的起点重新开始计算。

第 2 轮：i = 1

当前下标变成 1。

先算当前站点的净收益。1 号站可以加 2 升油，从 1 号站开到 2 号站要花 4 升油，所以这个站点的净收益也是负 2。

把负 2 累加到全局总油量里。上一轮结束后 `total` 是负 2，再加上负 2，`total` 变成负 4。

再把负 2 累加到当前路径剩余油量里。因为上一轮已经重置过，所以现在的 `tank` 是 0，0 加上负 2 以后，`tank` 变成负 2。

这说明如果把 1 号站当作起点，走到当前下标 1 时也已经走不下去了。

所以 1 号站也不能作为起点。

把下一个站，也就是 2 号站，设为新的候选起点，所以 `start` 变成 2。

再一次把 `tank` 重置为 0。

第 3 轮：i = 2

当前下标变成 2。

先算当前站点的净收益。2 号站可以加 3 升油，从 2 号站开到 3 号站要花 5 升油，所以这个站点的净收益还是负 2。

把负 2 累加到全局总油量里。原来的 `total` 是负 4，再加上负 2，`total` 变成负 6。

再把负 2 累加到当前路径剩余油量里。当前的 `tank` 是 0，加上负 2 以后，`tank` 变成负 2。

现在 `tank` 又小于 0，说明从 2 号站出发，走到当前下标 2 时也已经走不下去了。

所以 2 号站还是不能作为起点。

把下一个站，也就是 3 号站，设为新的候选起点，所以 `start` 变成 3。

同时把 `tank` 重置为 0。

第 4 轮：i = 3

当前下标变成 3。

先算当前站点的净收益。3 号站可以加 4 升油，从 3 号站开到 4 号站要花 1 升油，所以这个站点的净收益是正 3。

把正 3 累加到全局总油量里。原来的 `total` 是负 6，加上正 3 以后，`total` 变成负 3。

再把正 3 累加到当前路径剩余油量里。原来的 `tank` 是 0，加上正 3 以后，`tank` 变成 3。

这一次 `tank` 没有小于 0，说明从候选起点 3 出发，至少走到当前下标 3 时是走得通的。

所以这一步不用修改 `start`，`start` 继续保持为 3。

第 5 轮：i = 4

当前下标变成 4。

先算当前站点的净收益。4 号站可以加 5 升油，从 4 号站开到 0 号站要花 2 升油，所以这个站点的净收益也是正 3。

把正 3 累加到全局总油量里。原来的 `total` 是负 3，加上正 3 以后，`total` 变成 0。

再把正 3 累加到当前路径剩余油量里。原来的 `tank` 是 3，加上正 3 以后，`tank` 变成 6。

现在 `tank` 依然没有小于 0，说明从候选起点 3 出发，继续走到当前下标 4 也没有问题。

循环到这里结束，`start` 还是 3。

循环结束后为什么返回 3

循环结束了，先看 `total`。现在 `total` 是 0，它没有小于 0，说明整个环路从总量上看，油量是够的。

再看 `start`。前面 0 号站、1 号站、2 号站都已经被一步一步排除了，最后保留下来的候选起点是 3。

所以最终返回 3。
