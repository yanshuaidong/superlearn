300. 最长递增子序列

Longest Increasing Subsequence — 动态规划 · 交互式教学

题目描述

给定一个整数数组 nums，找到其中最长严格递增子序列的长度。

子序列是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。

例如，[3,6,2,7] 是数组 [0,3,1,6,2,2,7] 的子序列。

示例：

输入：nums = [10,9,2,5,3,7,101,18]

输出：4

解释：最长递增子序列是 [2,3,7,18]，长度为 4。

关键思路：为什么用 DP？

核心定义：dp[i] = 以 nums[i] 结尾的最长严格递增子序列的长度。

想要找整个数组的最长递增子序列，最直觉的思路是：枚举所有子序列，检查哪些是递增的，然后找最长的。

但子序列总数是 2n，太慢了。DP 的智慧在于：

我不需要真的枚举所有子序列！对于位置 i，我只关心"以 nums[i] 结尾的最长递增子序列有多长"。 而这个信息，可以从前面已经算好的 dp[0], dp[1], ..., dp[i-1] 中推导出来。

状态转移公式

对于每个 i，枚举所有 j（0 ≤ j < i）：

如果 nums[j] < nums[i]，则 dp[i] = Math.max(dp[i], dp[j] + 1)

最终答案 = max(dp[0], dp[1], ..., dp[n-1])

公式背后的暴力枚举视角

dp[j] + 1 到底在干什么？

dp[j] 已经浓缩了"以 nums[j] 结尾的所有递增子序列"中最长那个的长度。

当 nums[j] < nums[i] 时，就等于说：

→ 把以 nums[j] 结尾的那个最长递增子序列拿过来

→ 在尾部追加 nums[i]（因为 nums[j] < nums[i]，仍然递增）

→ 新子序列长度 = dp[j] + 1

对所有满足条件的 j 都试一遍，取最大值，就得到了 dp[i]。

这就是 DP 在帮我们做的事情 —— 它把暴力枚举 2n 个子序列的工作，压缩成了 O(n2) 的比较。

答案代码（带详细注释）

var lengthOfLIS = function(nums) {
    // 边界处理：空数组直接返回 0
    if (nums.length === 0) return 0;

    // 创建 dp 数组，全部初始化为 1
    // 为什么是 1？因为每个元素自身就是一个长度为 1 的递增子序列
    const dp = new Array(nums.length).fill(1);

    // maxLen 记录全局最大值，初始为 1（至少有一个元素）
    let maxLen = 1;

    // 外层循环：依次计算每个位置 i 的 dp[i]
    for (let i = 0; i < nums.length; i++) {

        // 内层循环：枚举 i 前面的每个位置 j
        // 这就是"暴力枚举所有可能的前驱"
        for (let j = 0; j < i; j++) {

            // 关键判断：nums[j] 必须严格小于 nums[i]
            // 只有这样，nums[i] 才能接到以 nums[j] 结尾的子序列后面
            if (nums[j] < nums[i]) {
                // 状态转移：dp[j]+1 表示"把 nums[i] 接到 j 的最长子序列后面"
                // 取 max 是因为可能有多个 j 都能接，我们要最长的那个
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }

        // 每算完一个 dp[i]，更新全局最大值
        // 因为最终答案不一定在最后一个位置，而是 dp 数组中的最大值
        maxLen = Math.max(maxLen, dp[i]);
    }

    return maxLen;
};
