模块一：题目与答案代码

题目怎么转成 DP

题目要求“连线不能相交”，这意味着匹配顺序必须一致。也就是说：

只要顺序必须保持一致，问题就从“画线”变成了“找公共子序列”。 所以这道题本质上就是两个数组的最长公共子序列长度。

状态定义

dp[i][j]：表示 nums1 前 i 个元素，和 nums2 前 j 个元素，最多能连多少条不相交的线。

因为多开了一行一列，所以访问真实元素时要写成 nums1[i - 1] 和 nums2[j - 1]。

转移思路

如果当前两个元素相等，就把它们连起来：dp[i][j] = dp[i - 1][j - 1] + 1。

如果不相等，就不能同时使用这两个元素，只能“跳过其中一个”：max(dp[i - 1][j], dp[i][j - 1])。

带中文注释的答案代码

var maxUncrossedLines = function(nums1, nums2) {
  // m 是第一个数组长度，n 是第二个数组长度
  const m = nums1.length;
  const n = nums2.length;

  // dp[i][j]：
  // nums1 前 i 个元素 与 nums2 前 j 个元素
  // 最多能连出的不相交线数量
  const dp = Array.from({ length: m + 1 }, () => Array(n + 1).fill(0));

  // i 和 j 从 1 开始，是因为第 0 行 / 第 0 列表示“有一边为空”
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      // 当前比较的是 nums1[i - 1] 和 nums2[j - 1]
      if (nums1[i - 1] === nums2[j - 1]) {
        // 相等：这两个元素可以新连一条线
        // 所以在左上角最优解基础上 +1
        dp[i][j] = dp[i - 1][j - 1] + 1;
      } else {
        // 不相等：当前这两个元素没法直接配对
        // 只能二选一：
        // 1. 跳过 nums1[i - 1]，看上方
        // 2. 跳过 nums2[j - 1]，看左方
        dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
      }
    }
  }

  // 整个数组的答案就在右下角
  return dp[m][n];
};

把“压缩状态”展开看

dp[i][j] 不是凭空来的，它代表“前缀内所有合法连线方案”的最优摘要。 这里把摘要重新拆开，看看本步到底比较了哪些候选。

左上角：同时使用当前两个元素
-
只有两个元素相等时，这条路才成立。
上方：跳过 nums1 当前元素
-
保留 nums2 的前 j 个元素不变。
左方：跳过 nums2 当前元素
-
保留 nums1 的前 i 个元素不变。

这一格背后的“暴力视角”

当你还没压缩成 DP 时，你其实会想：

“当前两个元素能不能连？如果不能，我是丢掉 nums1 的当前元素，还是丢掉 nums2 的当前元素？”

DP 只是把这些分支的最优答案存进表格里，方便下次直接复用。
