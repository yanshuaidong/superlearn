121. 买卖股票的最佳时机

动态规划 · 二维状态定义 · 只允许一次交易

题目描述

给定一个数组 prices，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择某一天买入这只股票，并选择在未来的某一个不同的日子卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0。

动态规划 数组 简单
示例：
输入：prices = [7, 1, 5, 3, 6, 4]
输出：5
解释：在第 2 天（价格 = 1）买入，在第 5 天（价格 = 6）卖出，利润 = 6 - 1 = 5。

核心思路：用 DP 模拟"每一天的决策"

关键问题：每一天结束时，我可能处于两种状态之一——手里有股票或手里没股票。
我们用 dp[i][0] 记录第 i 天结束时"不持有股票"的最大利润，
用 dp[i][1] 记录第 i 天结束时"持有股票"的最大利润。

为什么要分两个状态？

因为到第 i 天结束时，"不持有股票"和"持有股票"是两种本质不同的局面。它们能做的后续操作不同，所以必须分开记录。

状态转移的暴力枚举本质

dp[i][0]（第 i 天不持有）本质上枚举的是：

① 前 i 天从未买过股票（利润为 0）

② 在第 0~i 天中的某一天买入、又在第 0~i 天中的某一天卖出，且卖出时间 ≥ 买入时间（利润 = 卖价 - 买价）

dp[i][0] 记录的就是这些情况里利润最大的那个。

dp[i][1]（第 i 天持有）本质上枚举的是：

在第 0~i 天中的某一天买入股票、至今没卖出，利润 = -买入价

dp[i][1] 记录的就是"在哪天买入使得花费最小（利润最大）"的那个值。

答案代码（含教学注释）

var maxProfit = function(prices) {
    // 边界处理：如果没有价格数据，无法交易
    if (!prices || prices.length === 0) return 0;

    const n = prices.length;

    // dp[i][0]：第 i 天结束时"不持有股票"的最大利润
    //   → 本质：枚举了"前 i 天所有可能的【买入→卖出】方案"的最大值，
    //     也包括"一直不买"这个方案（利润为 0）
    // dp[i][1]：第 i 天结束时"持有股票"的最大利润
    //   → 本质：枚举了"在第 0~i 天中哪天买入"的最优方案，
    //     利润 = -买入价（因为只能买一次，买入前利润必为 0）
    const dp = new Array(n).fill(0).map(() => new Array(2).fill(0));

    // 第 0 天：如果不持有 → 利润为 0（还没买过）
    //         如果持有   → 必定是今天买的，花了 prices[0]
    dp[0][0] = 0;
    dp[0][1] = -prices[0];

    for (let i = 1; i < n; i++) {
        // 今天不持有：要么"昨天就不持有"，要么"昨天持有+今天卖掉"
        // 暴力视角：把 dp[i-1][0] 里的所有方案照搬过来，
        //   再加上"把 dp[i-1][1] 代表的最优持有 + 今天卖出"这个新方案
        dp[i][0] = Math.max(dp[i - 1][0], dp[i - 1][1] + prices[i]);

        // 今天持有：要么"昨天就持有"，要么"今天新买入"
        // 暴力视角：dp[i-1][1] 里的所有"第 0~i-1 天买入"方案照搬过来，
        //   再加上"今天第 i 天买入"这个新方案（利润 = -prices[i]）
        // 两者取最大值 = 花费最少的买入方案
        dp[i][1] = Math.max(dp[i - 1][1], -prices[i]);
    }

    // 最终答案：最后一天不持有的最大利润
    // 因为持有股票说明还没卖，利润未实现
    return dp[n - 1][0];
};

状态转移公式总结

dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])

含义A dp[i-1][0]：昨天就不持有 → 今天继续不持有
含义B dp[i-1][1] + prices[i]：昨天持有 → 今天卖出
dp[i][1] = max(dp[i-1][1], -prices[i])

含义A dp[i-1][1]：昨天就持有 → 今天继续持有
含义B -prices[i]：今天新买入（只允许买一次！）
注意 dp[i][1] 的买入项是 -prices[i] 而不是 dp[i-1][0] - prices[i]！
因为题目只允许买卖一次，所以买入前的利润一定是 0，不可能有之前卖出的利润。这是本题和"多次买卖"变体的核心区别。
