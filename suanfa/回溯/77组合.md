结论：这题用 **回溯（DFS）** 最合适。
核心思路是：从 `1 ~ n` 里依次尝试选数，把当前选中的数放进路径里；当路径长度等于 `k` 时，就得到一个合法组合，加入答案。

下面是 **JavaScript 代码**，并附带 **详细中文注释**：

```javascript
/**
 * 77. 组合
 * 给定两个整数 n 和 k，返回范围 [1, n] 中所有可能的 k 个数的组合。
 *
 * 解题思路：
 * 使用回溯（Backtracking）。
 *
 * 例如 n = 4, k = 2：
 * 我们需要从 [1,2,3,4] 中选出 2 个数的所有组合。
 *
 * 回溯本质：
 * - 每一步决定“当前选哪个数”
 * - 把选择加入路径 path
 * - 递归继续往后选
 * - 递归结束后撤销选择（回溯）
 *
 * 组合和排列的区别：
 * - 组合不关心顺序，所以比如 [1,2] 和 [2,1] 视为同一种
 * - 因此递归时，下一层只能从“当前数字的后面”开始选
 *
 * 时间复杂度：
 * - 大致为 O(C(n, k) * k)
 * - 因为一共有 C(n, k) 个组合，每次复制 path 需要 O(k)
 *
 * 空间复杂度：
 * - 递归栈深度 O(k)
 * - 不计答案数组时，额外空间为 O(k)
 *
 * @param {number} n
 * @param {number} k
 * @return {number[][]}
 */
var combine = function(n, k) {
    // 最终返回的结果数组
    const res = [];
    
    // path 用来存放当前已经选中的数字组合
    const path = [];

    /**
     * 回溯函数
     * @param {number} start 当前这一层从哪个数字开始选择
     */
    function backtrack(start) {
        // 终止条件：
        // 如果当前路径长度已经等于 k，说明找到一个合法组合
        if (path.length === k) {
            // 注意要拷贝一份 path，因为后续 path 还会继续变化
            res.push([...path]);
            return;
        }

        /**
         * 剪枝优化：
         * 当前还需要选的个数 = k - path.length
         *
         * 假设当前从 i 开始选，
         * 那么后面可用的数字个数必须 >= 还需要选的个数
         *
         * 所以 i 最大可以到：
         * n - (k - path.length) + 1
         *
         * 举例：
         * n = 4, k = 2
         * 如果当前 path.length = 0，还需要选 2 个
         * 那么 i 最大只能到 3
         * 因为如果 i=4，后面已经不够凑出 2 个数了
         */
        for (let i = start; i <= n - (k - path.length) + 1; i++) {
            // 做选择：把当前数字 i 加入路径
            path.push(i);

            // 递归进入下一层，只能从 i+1 开始选，避免重复组合
            backtrack(i + 1);

            // 撤销选择（回溯），恢复现场
            path.pop();
        }
    }

    // 从数字 1 开始回溯
    backtrack(1);

    return res;
};
```

# 一、先解释关键变量 / 状态分别表示什么

这段代码里，最关键的变量和状态有这几个：

## 1）`n`

表示可选数字的范围上限。
也就是说，我们可以从 `1 ~ n` 这些数字里选。

比如：

* `n = 4`，可选数字就是 `1, 2, 3, 4`
* `n = 1`，可选数字就是 `1`

---

## 2）`k`

表示每个组合里，最终要选几个数字。

比如：

* `k = 2`，说明每个组合必须正好有 2 个数
* `k = 1`，说明每个组合必须正好有 1 个数

---

## 3）`res`

表示**最终结果数组**。
它用来存放所有已经找到的合法组合。

例如最后可能变成：

```js
[
  [1,2],
  [1,3],
  [1,4],
  [2,3],
  [2,4],
  [3,4]
]
```

一开始它是空数组：

```js
[]
```

---

## 4）`path`

表示**当前递归路径上已经选中的数字**。
你可以把它理解成：“正在尝试拼出来的那个组合”。

比如：

* `path = [1]` 表示当前已经选了数字 `1`
* `path = [1,3]` 表示当前这条路径选中了 `1` 和 `3`
* 当 `path.length === k` 时，说明已经凑够一个完整组合了

一开始它也是空数组：

```js
[]
```

---

## 5）`backtrack(start)`

这是回溯函数，表示：

> 当前这一层，接下来只能从 `start` 开始往后选数字。

为什么要有 `start`？

因为这是“组合”不是“排列”，顺序不重要。
所以一旦选了 `i`，下一层只能从 `i+1` 开始继续选，避免重复。

例如：

* 已经选了 `1`，下一层就只能从 `2` 开始选
* 这样会得到 `[1,2]`，但不会再去得到 `[2,1]`

---

## 6）`if (path.length === k)`

这是递归终止条件。含义是：

> 如果当前已经选满了 `k` 个数字，那当前 `path` 就是一个合法组合。

这时会执行：

```js
res.push([...path]);
return;
```

注意这里必须用 `[...path]` 拷贝一份。
因为 `path` 后面还会继续 `pop()` 回退，如果不拷贝，存进 `res` 的内容会被后续修改掉。

---

## 7）`for (let i = start; i <= n - (k - path.length) + 1; i++)`

这是当前层尝试选择数字的循环。

这里上界不是直接写 `n`，而是用了剪枝：

```js
n - (k - path.length) + 1
```

它的含义是：

* `k - path.length` = 当前还需要再选几个数
* 从 `i` 开始往后，至少要剩下这么多个数，才有机会凑满一个组合
* 如果后面的数已经不够了，就没必要继续循环了

这个上界的作用是：**提前排除不可能成功的分支**

# 四、示例 1 整个过程结束后，为什么返回这个结果

最后 `res` 里是：

```js
[
  [1,2],
  [1,3],
  [1,4],
  [2,3],
  [2,4],
  [3,4]
]
```

为什么是这个结果？

因为代码按照下面的真实顺序，把所有长度为 `2` 的合法 `path` 都收集到了：

1. 先固定第一个数为 `1`，再依次尝试后面的 `2,3,4`

   * 得到 `[1,2]`
   * 得到 `[1,3]`
   * 得到 `[1,4]`

2. 再固定第一个数为 `2`，再依次尝试后面的 `3,4`

   * 得到 `[2,3]`
   * 得到 `[2,4]`

3. 再固定第一个数为 `3`，后面只剩 `4`

   * 得到 `[3,4]`

4. 第一个数不会取 `4`，因为剪枝已经提前判断出：如果第一个数是 `4`，后面不够再选 1 个数，凑不出长度为 `2` 的组合

所以最终返回这些组合，而且顺序就是代码实际 DFS / 回溯的访问顺序。

# 八、补一层“这段代码到底是怎么工作的”

你可以把它理解成下面这个过程：

1. `path` 是“当前正在试的组合”
2. 每层 `for` 循环决定当前这一位选哪个数
3. 选了之后就递归去填下一位
4. 如果 `path.length === k`，说明已经拼成一个完整组合，存进 `res`
5. 递归返回后，用 `pop()` 把刚才选的数撤销，再换一个数继续试
6. `start` 保证后面的数只能往后选，所以不会出现重复组合
7. 剪枝上界保证“不够凑满”的情况直接不试，提高效率
