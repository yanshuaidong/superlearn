结论：这道题用**贪心算法**最合适。
核心思路是：**按区间右端点从小到大排序，尽量优先保留结束更早的区间**，这样才能给后面的区间留下更多空间。最后用“总区间数 - 最多能保留的不重叠区间数”得到最少移除数量。

---

## 一、题目思路

我们要求的是：

* **最少移除多少个区间**
* 使得剩下的区间**互不重叠**

这个问题可以转换为：

* **最多能保留多少个互不重叠的区间**

那么答案就是：

```js
最少移除数量 = 总区间数 - 最多保留数量
```

---

## 二、为什么按右端点排序？

假设两个区间冲突了，我们应该优先保留谁？

当然应该保留**结束更早**的那个，因为它占用的空间更少，更有利于后面接更多区间。

所以做法是：

1. 按区间的 **end 升序排序**
2. 先选第一个区间
3. 之后遍历每个区间：

   * 如果当前区间的 `start >= 上一个被选中区间的 end`
   * 说明不重叠，可以保留
4. 统计最多能保留多少个区间

---

## 三、举例说明

### 示例 1

```js
intervals = [[1,2],[2,3],[3,4],[1,3]]
```

按右端点排序后：

```js
[[1,2],[2,3],[1,3],[3,4]]
```

选取过程：

* 先选 `[1,2]`
* `[2,3]` 的开始点 `2 >= 2`，不重叠，选
* `[1,3]` 的开始点 `1 < 3`，重叠，不选
* `[3,4]` 的开始点 `3 >= 3`，不重叠，选

最多保留 3 个区间，总共 4 个区间，所以最少移除：

```js
4 - 3 = 1
```

## 四、JavaScript 代码（带详细中文注释）

```javascript
/**
 * 435. 无重叠区间
 * 
 * 题意：
 * 给定若干区间，求最少移除多少个区间，才能让剩下的区间互不重叠。
 * 
 * 贪心思路：
 * 1. 按区间右端点（结束位置）从小到大排序
 * 2. 尽可能选择结束早的区间，这样可以为后续区间腾出更多空间
 * 3. 统计最多能保留多少个不重叠区间
 * 4. 最少移除数量 = 总区间数 - 最多保留数量
 * 
 * 注意：
 * [1,2] 和 [2,3] 不算重叠，因为它们只是端点接触
 * 所以判断条件是：当前区间 start >= 上一个保留区间 end
 * 
 * @param {number[][]} intervals
 * @return {number}
 */
var eraseOverlapIntervals = function(intervals) {
    // 如果区间数量为 0 或 1，肯定不需要移除
    if (intervals.length <= 1) {
        return 0;
    }

    // 按区间的右端点从小到大排序
    // 如果结束位置相同，谁前谁后都无所谓
    intervals.sort((a, b) => a[1] - b[1]);

    // count 表示当前最多能保留多少个不重叠区间
    let count = 1;

    // end 表示上一个被保留区间的结束位置
    let end = intervals[0][1];

    // 从第二个区间开始遍历
    for (let i = 1; i < intervals.length; i++) {
        let start = intervals[i][0];

        // 如果当前区间的起点 >= 上一个保留区间的终点
        // 说明这两个区间不重叠，可以保留当前区间
        if (start >= end) {
            count++;
            end = intervals[i][1];
        }
        // 否则重叠，直接跳过当前区间
        // 因为我们已经按右端点排序了，当前留下的方案一定更优
    }

    // 最少移除数量 = 总区间数 - 最多保留数量
    return intervals.length - count;
};
```

* 遍历：`O(n)`


### 空间复杂度

## 六、为什么这个贪心是对的？

因为每次都优先选择**结束最早**的区间，不会妨碍后面选择更多区间。

这和经典题 **“活动选择问题”** 是同一种模型：

* 想要选最多互不重叠区间
* 最优策略就是每次选结束最早的那个

所以最终得到的保留数量一定是最大值，从而移除数量最小。

所以函数最终返回：

```js
1
```

---

# 六、整个过程结束后，为什么返回这个结果

因为这段代码实际做的是两件事：

## 第 1 件事：找出最多能保留多少个不重叠区间

通过排序和遍历，代码实际保留下来的区间是：

* `[1,2]`
* `[2,3]`
* `[3,4]`

一共保留了：

```js
count = 3
```

---

## 第 2 件事：用“总数 - 保留数”算出最少移除数

总区间数是：

```js
intervals.length = 4
```

最多能保留：

```js
count = 3
```

所以最少需要移除：

```js
4 - 3 = 1
```

---

## 为什么被移除的是 `[1,3]`？

因为在排序后遍历过程中，轮到 `[1,3]` 时：

* 它的起点 `1`
* 小于当前 `end = 3`

所以判断：

```js
start >= end
```

不成立。

这说明它和当前已经保留的区间发生了重叠，因此不能加入保留集合。
所以它被“跳过”，本质上就相当于被移除了。

---

## 最终因果链条可以概括成一句人话

代码先把结束早的区间放前面，然后从前往后扫：

* 先保留 `[1,2]`
* `[2,3]` 能接上，保留
* `[1,3]` 和前面冲突，跳过
* `[3,4]` 能接上，保留

最后 4 个区间里保住了 3 个，所以只需要移除 1 个，返回 `1`。
