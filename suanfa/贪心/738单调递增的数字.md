结论：这题用**贪心**最合适。
核心思路是：**一旦发现某一位比后一位大，就把这一位减 1，并把它后面的所有位都变成 9**。为了处理连锁反应，需要继续往前检查。

---

## 一、题意理解

如果一个数字从左到右每一位都满足：

* 前一位 `<=` 后一位

那么这个数字就是“单调递增”的。

例如：

* `1234` 是单调递增
* `11239` 是单调递增
* `332` 不是，因为 `3 > 2`

现在要求：

* 在所有 `<= n` 的数字中
* 找到**最大的**
* 且这个数字的各位数字单调递增

---

## 二、贪心思路

以 `n = 332` 为例：

### 1. 先看哪里破坏了单调性

`332`

从左到右看：

* `3 <= 3`，没问题
* `3 <= 2`，出问题了

说明当前这个数不能直接用。

---

### 2. 怎么变成尽可能大？

既然第二个 `3 > 2`，为了让数字变小且尽可能大：

* 把前一个出问题的数字减 `1`
* 后面全部变成 `9`

即：

* `332`
* 第二个 `3` 减 1 变成 `2`
* 后面变 9
* 得到 `329`

但 `329` 仍然不是单调递增，因为：

* `3 > 2`

所以还要继续往前处理：

* 第一个 `3` 再减 1 变成 `2`
* 后面全变 9
* 得到 `299`

`299` 就满足单调递增了。

---

## 三、为什么后面都变成 9？

因为题目要求的是：

* **不超过 n**
* **尽可能大**

当前某一位已经必须减小了，那为了让结果最大，后面的位当然应该都尽量取最大值，也就是 `9`。

---

## 四、算法步骤

把数字转成字符数组 `arr`：

1. 从右往左遍历
2. 如果发现 `arr[i - 1] > arr[i]`

   * 说明这里破坏了单调递增
   * 就把 `arr[i - 1]` 减 1
   * 记录位置 `i`
3. 最后把记录位置之后的所有位都改成 `'9'`
4. 转回数字返回

---

## 五、代码（JavaScript，带详细中文注释）

```javascript
/**
 * @param {number} n
 * @return {number}
 */
var monotoneIncreasingDigits = function(n) {
    // 将数字转成字符数组，方便逐位处理
    let arr = String(n).split('');
    
    // mark 用来记录：从哪个位置开始，后面的数字都应该变成 9
    // 默认设为数组长度，表示如果一直没有出现逆序，就不用改
    let mark = arr.length;
    
    // 从右向左遍历
    // 为什么从右往左？
    // 因为一旦某一位需要减 1，可能会继续影响它左边的数字
    for (let i = arr.length - 1; i > 0; i--) {
        // 如果前一位 > 当前位，说明不是单调递增
        if (arr[i - 1] > arr[i]) {
            // 把前一位减 1
            // 字符需要先转成数字，再减，再转回字符
            arr[i - 1] = String(Number(arr[i - 1]) - 1);
            
            // 记录这个位置
            // 从这个位置开始，后面的所有位最后都要变成 9
            mark = i;
        }
    }
    
    // 把 mark 及其后面的所有位都变成 9
    for (let i = mark; i < arr.length; i++) {
        arr[i] = '9';
    }
    
    // 将字符数组拼接成字符串，再转成数字返回
    return Number(arr.join(''));
};
```

# 十二、整个过程结束后，为什么返回这个结果

现在单独解释最后这个结论，而且要和执行过程对应起来。

原数是：

```js
332
```

代码从右往左检查：

1. 先发现下标 1 和 2 这里有问题：`3 > 2`

   * 把前一个 `3` 减成 `2`
   * `arr` 从 `['3','3','2']` 变成 `['3','2','2']`
   * 同时记录 `mark = 2`

2. 再继续往左检查，发现下标 0 和 1 这里也有问题：`3 > 2`

   * 把前一个 `3` 再减成 `2`
   * `arr` 从 `['3','2','2']` 变成 `['2','2','2']`
   * 同时把 `mark` 更新成 `1`

3. 第一个循环结束后，说明：

   * 从位置 `1` 开始，后面的位都应该改成 `9`
   * 于是第二个循环把 `arr[1]` 和 `arr[2]` 都改成 `'9'`
   * 最终得到 `['2','9','9']`

4. 拼接并转数字后得到：

```js
299
```

---

## 为什么这个结果是对的

因为题目要求的是：

* 结果必须 `<= 332`
* 各位数字还要满足单调递增
* 并且要尽可能大

`299` 满足这三点：

### 1. 它不超过 332

```js
299 <= 332
```

### 2. 它是单调递增的

从左到右看：

```js
2 <= 9 <= 9
```

成立。

### 3. 它已经尽可能大

因为一旦发现某一位太大，必须把它减小；
而只要前面已经减小了，后面为了让整体尽量大，最优做法就是全部填 `9`。

所以最终返回 `299`。
