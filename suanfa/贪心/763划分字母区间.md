思路：**先记录每个字符最后一次出现的位置，再贪心地扫描字符串，尽可能早地切分区间。**

下面我先讲清楚做法，再给你完整的 JavaScript 代码（带详细中文注释）。

---

## 一、题目理解

题目要求把字符串 `s` 划分成尽可能多的片段，并且满足：

* 同一个字母 **最多只出现在一个片段里**
* 所有片段按原顺序拼接后，仍然等于原字符串

也就是说：

* 如果某个字符在后面还会出现，那么当前片段就**不能提前结束**
* 只有当当前片段里所有字符的“最后出现位置”都已经被包含进来了，才能切分

---

## 二、核心思路

### 1）先统计每个字符最后出现的位置

例如：

```js
s = "ababcbacadefegdehijhklij"
```

我们先遍历一遍字符串，记录每个字符最后一次出现的下标。

比如：

* `a` 最后出现在 8
* `b` 最后出现在 5
* `c` 最后出现在 7
* ...

---

### 2）再遍历字符串，动态维护当前片段的最远边界

定义：

* `start`：当前片段的起始位置
* `end`：当前片段目前必须覆盖到的最远位置

遍历到位置 `i` 时：

* 看当前字符 `s[i]` 的最后出现位置
* 用它来更新 `end`

即：

```js
end = Math.max(end, last[s[i]])
```

含义是：

* 当前片段中出现了 `s[i]`
* 那么这个片段至少要覆盖到 `s[i]` 最后一次出现的位置
* 所以片段右边界要不断向右扩展

---

### 3）什么时候可以切分？

当遍历到某个位置 `i`，发现：

```js
i === end
```

说明：

* 当前片段中所有字符，它们最后一次出现的位置都不超过 `i`
* 也就是说这个片段已经完整了
* 可以在这里切一刀

片段长度就是：

```js
end - start + 1
```

然后开启下一段：

```js
start = i + 1
```

---

## 三、为什么这是贪心？

因为题目要求“划分尽可能多的片段”。

那我们就应该：

* **一旦当前片段可以结束，就立刻结束**
* 这样才能让后面保留更多机会继续切分

这就是贪心策略。

## 四、代码（JavaScript，带详细中文注释）

```javascript
/**
 * 763. 划分字母区间
 * 
 * 解题思路：
 * 1. 先遍历字符串，记录每个字符最后一次出现的位置
 * 2. 再遍历字符串，维护当前片段的右边界 end
 * 3. 当遍历下标 i 到达 end 时，说明当前片段可以切分
 * 
 * 时间复杂度：O(n)
 * 空间复杂度：O(1) / O(26~字符集大小)
 * 这里如果只考虑小写字母，可以认为是常数空间
 * 
 * @param {string} s
 * @return {number[]}
 */
var partitionLabels = function(s) {
    // 用来记录每个字符最后一次出现的位置
    // 例如：last['a'] = 8，表示字符 'a' 最后一次出现在下标 8
    const last = {};

    // 第一次遍历：统计每个字符最后一次出现的位置
    for (let i = 0; i < s.length; i++) {
        last[s[i]] = i;
    }

    // 结果数组，存放每个片段的长度
    const result = [];

    // start 表示当前片段的起始位置
    let start = 0;

    // end 表示当前片段目前“必须延伸到”的最远位置
    let end = 0;

    // 第二次遍历：贪心划分区间
    for (let i = 0; i < s.length; i++) {
        // 当前字符最后出现的位置，可能会把当前片段的右边界继续往后推
        end = Math.max(end, last[s[i]]);

        // 如果当前位置 i 已经走到了当前片段的最远右边界
        // 说明当前片段中的所有字符都不会再在后面出现了
        // 因此可以在这里切分出一个合法片段
        if (i === end) {
            // 当前片段长度 = 右边界 - 左边界 + 1
            result.push(end - start + 1);

            // 下一段从 i + 1 开始
            start = i + 1;
        }
    }

    return result;
};
```

## 六、总结

这题的关键点就一句话：

> **一个片段的右边界，必须覆盖该片段中所有字符的最后出现位置。**

所以做法是：

* 先预处理每个字符最后出现的位置
* 再贪心扫描，维护当前片段最远右边界
* 当扫描位置到达右边界时，立即切分

# 四、第二遍遍历：真正进行划分

这部分对应代码：

```js
const result = [];
let start = 0;
let end = 0;

for (let i = 0; i < s.length; i++) {
    end = Math.max(end, last[s[i]]);

    if (i === end) {
        result.push(end - start + 1);
        start = i + 1;
    }
}
```

## 第二遍开始前，变量初始值

```js
result = []
start = 0
end = 0
```

含义是：

* 目前还没有切出任何片段，所以 `result` 是空的
* 当前片段从下标 `0` 开始，所以 `start = 0`
* 当前片段的右边界还没真正确定，先设成 `0`

---

## 第 1 轮（第二遍遍历）

当前 `i = 0`
当前读到 `s[0] = 'a'`

先执行：

```js
end = Math.max(end, last[s[i]])
```

也就是：

```js
end = Math.max(0, last['a'])
end = Math.max(0, 8)
end = 8
```

更新前：

* `end = 0`

更新后：

* `end = 8`

为什么变成 8？
因为当前字符是 `'a'`，而 `'a'` 最后一次出现的位置在 8。
既然当前片段里已经包含了 `'a'`，那这个片段至少要延伸到 8，不然前面的 `'a'` 和后面的 `'a'` 就会被分开。

然后判断：

```js
if (i === end)
```

也就是判断：

```js
0 === 8
```

结果是 **false**，所以**不进入 if**。

为什么不进入？
因为现在才走到下标 0，还没走到当前片段必须覆盖的最远位置 8，所以还不能切分。

这一步说明：
当前片段暂时至少要覆盖区间 `[0, 8]`。

---

## 第 2 轮

当前 `i = 1`
当前读到 `s[1] = 'b'`

执行：

```js

     = Math.max(8, 5)
     = 8
```

更新前：

* `end = 8`

更新后：

* `end = 8`

没有变化。

为什么没变化？
因为 `'b'` 的最后位置是 5，比当前 `end = 8` 更靠左，所以不需要再把右边界往后推。

然后判断：

```js
1 === 8
```

# 六、整个过程结束后，为什么返回这个结果

最后返回 `[9, 7, 8]`，原因不是“猜出来的”，而是代码按顺序执行后自然得到的。

核心因果关系是这样的：

## 第一个片段为什么是长度 9

从下标 `0` 开始时，读到 `'a'`，发现 `'a'` 最后一次出现在 `8`，所以当前片段至少要到 `8`。
后面继续读 `'b'`、`'c'` 等字符，它们的最后位置分别是 `5`、`7`，都没有超过 `8`。
直到 `i = 8` 时，终于走到了当前片段的最远边界，所以切出第一段：

* 区间 `[0, 8]`
* 长度 `8 - 0 + 1 = 9`

---

## 第二个片段为什么是长度 7

下一段从 `9` 开始。
读到 `'d'` 时，`end` 变成 `14`；
读到 `'e'` 时，`end` 又被推到 `15`；
后面 `'f'`、`'g'`、`'d'` 都没有把 `end` 再往后推。
直到 `i = 15`，到达边界，于是切出第二段：

* 区间 `[9, 15]`
* 长度 `15 - 9 + 1 = 7`

---

## 第三个片段为什么是长度 8

下一段从 `16` 开始。
读到 `'h'` 时，`end = 19`；
读到 `'i'` 时，`end = 22`；
读到 `'j'` 时，`end = 23`。
后面 `'k'`、`'l'` 等都没有再把边界推远。
直到 `i = 23`，到达边界，切出第三段：

* 区间 `[16, 23]`
* 长度 `23 - 16 + 1 = 8`

---

## 为什么这就是正确结果

因为每次切分都发生在 `i === end` 的时刻。
这意味着：当前片段里出现过的所有字符，它们最后一次出现位置都已经落在这个片段内部，不会跑到后面的片段里。

所以得到的三个片段：

* `"ababcbaca"`
* `"defegde"`
* `"hijhklij"`

恰好满足题目要求：
**每个字母最多只出现在一个片段中。**

并且代码是一旦能切就立刻切，也就是尽早切分，所以得到的是题目想要的划分方式，长度数组就是：

```js
[9, 7, 8]
```
