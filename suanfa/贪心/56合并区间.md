结论：这题的核心做法是**先按区间左端点排序，再依次合并**。
只要当前区间的起点 `<=` 已合并区间的终点，就说明有重叠，更新终点即可；否则就开启一个新区间。

下面是 **JavaScript 代码 + 详细中文注释**：

```javascript
/**
 * 56. 合并区间
 * 
 * 思路：
 * 1. 先按照每个区间的起点从小到大排序
 * 2. 准备一个结果数组 result
 * 3. 遍历排序后的区间：
 *    - 如果 result 为空，直接放进去
 *    - 如果当前区间和 result 最后一个区间重叠，则合并
 *    - 如果不重叠，则直接加入 result
 * 
 * 重叠判断：
 * 当前区间的 start <= 结果数组最后一个区间的 end
 * 说明两个区间有交集，或者首尾相接（题目中 [1,4] 和 [4,5] 也算重叠）
 * 
 * 时间复杂度：O(n log n)
 * - 排序耗时 O(n log n)
 * - 遍历合并耗时 O(n)
 * 
 * 空间复杂度：O(n)
 * - 结果数组最坏情况下要存放所有区间
 * 
 * @param {number[][]} intervals
 * @return {number[][]}
 */
var merge = function(intervals) {
    // 如果数组长度小于等于 1，说明本身就不需要合并，直接返回
    if (intervals.length <= 1) {
        return intervals;
    }

    // 第一步：按照区间起点升序排序
    // 比如 [[4,7],[1,4]] 会变成 [[1,4],[4,7]]
    intervals.sort((a, b) => a[0] - b[0]);

    // 用于存放最终合并后的区间
    const result = [];

    // 先把第一个区间放进去，作为初始区间
    result.push(intervals[0]);

    // 从第二个区间开始遍历
    for (let i = 1; i < intervals.length; i++) {
        // 当前要处理的区间
        const current = intervals[i];

        // result 中最后一个区间，表示“当前已经合并好的最后区间”
        const last = result[result.length - 1];

        // 判断是否重叠
        // 如果 current 的起点 <= last 的终点，说明重叠
        if (current[0] <= last[1]) {
            // 合并区间：
            // 起点不用变，因为 last 的起点一定更小或相等（已经排序）
            // 终点取两者较大值
            last[1] = Math.max(last[1], current[1]);
        } else {
            // 如果不重叠，说明当前区间应该作为一个新区间加入结果
            result.push(current);
        }
    }

    // 返回合并后的区间数组
    return result;
};
```

 * @param {number[][]} intervals
 * @return {number[][]}

var merge = function(intervals) {
    if (intervals.length <= 1) return intervals;

    intervals.sort((a, b) => a[0] - b[0]);

    const result = [intervals[0]];

    for (let i = 1; i < intervals.length; i++) {
        const current = intervals[i];

## 一、关键变量 / 状态分别表示什么

先把代码里几个最重要的变量说清楚，不然后面容易看乱。

### 1. `intervals`

这是输入的区间数组。
里面每个元素都是一个长度为 2 的数组，比如 `[1,3]`，表示一个区间：起点是 `1`，终点是 `3`。

例如：

```js

```

表示一共有 4 个区间。

---

### 2. `result`

这是结果数组。

可以把它理解成：

* `result` 里装的是“目前已经处理完，并整理好的区间”
* 特别是 `result` 的最后一个区间，代表“当前正在和后面区间比较的那个合并结果”

---

### 3. `i`

这是 `for` 循环里的下标，表示：

* 现在处理的是 `intervals[i]`

因为代码先把第一个区间放进 `result` 了，所以 `i` 是从 `1` 开始，不是从 `0` 开始。

---

### 4. `current`

这是当前轮正在处理的区间：

```js

```

也就是这一次循环读到的区间。

---

### 5. `last`

这是 `result` 里面最后一个区间：

```js
const last = result[result.length - 1];
```

它表示：

* 当前已经合并好的最后一个区间
* 当前要拿来和 `current` 做比较的区间

---

### 6. `current[0] <= last[1]`

这是判断是否重叠的条件。

含义是：

* `current[0]`：当前区间的起点
* `last[1]`：结果数组最后一个区间的终点

如果当前区间的起点，小于等于最后一个已合并区间的终点，就说明两个区间连上了，或者重叠了，就要合并。

---

### 7. `last[1] = Math.max(last[1], current[1])`

这是合并时真正执行的更新。

意思是：

* 起点不改，仍然用 `last[0]`
* 终点取更大的那个

也就是把两个区间合成一个更大的区间。

## 最后用一句人话总结这个执行过程

* 先拿第一个区间当作“当前已合并区间”
* 后面的每个区间，依次去和 `result` 最后一个区间比较

* 最后 `result` 里剩下的，就是所有合并后的区间
