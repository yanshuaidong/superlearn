结论：这题用**二叉树后序遍历 + 贪心**最好做。

核心思想是：
**优先在“孩子需要被监控”的父节点上放摄像头**，这样一个摄像头可以同时覆盖“父节点、自己、左右孩子”，覆盖范围最大。

---

## 一、思路分析

我们给每个节点设计 3 种状态：

* **0：该节点没有被监控**
* **1：该节点放了摄像头**
* **2：该节点已经被监控，但自己没有摄像头**

然后使用 **后序遍历**（先处理左右子树，再处理当前节点），因为一个节点是否要放摄像头，取决于它的左右孩子状态。

---

## 二、状态转移规则

假设当前节点是 `node`，左右孩子状态分别是 `left` 和 `right`。

### 情况 1：只要有一个孩子没有被监控

也就是：

* `left === 0` 或 `right === 0`

那当前节点 **必须放摄像头**，否则这个没被监控的孩子没人管。

于是：

* 当前节点状态 = `1`
* 摄像头数量 +1

---

### 情况 2：如果左右孩子中有一个放了摄像头

也就是：

* `left === 1` 或 `right === 1`

那么当前节点一定已经被监控到了。

于是：

* 当前节点状态 = `2`

---

### 情况 3：左右孩子都已经被监控，但都没有摄像头

也就是：

* `left === 2 && right === 2`

此时当前节点**没有被监控**，因为孩子没有摄像头，覆盖不到父节点。

于是：

* 当前节点状态 = `0`

---

## 三、为什么空节点要返回“已被监控”？

空节点不需要摄像头，也不需要被监控。
为了让叶子节点的判断更自然，我们规定：

* `null` 节点返回 `2`（已被监控）

这样叶子节点的左右孩子都是 `2`，叶子节点自己就会变成 `0`（未被监控），从而促使它的父节点来放摄像头，这正是贪心策略想要的效果。

---

## 四、为什么最后还要单独判断根节点？

因为根节点没有父节点。
如果整棵树递归结束后，根节点状态还是 `0`（未被监控），那就只能在根节点再补一个摄像头。

## 五、JavaScript 代码（带详细中文注释）

```javascript
/**
 * Definition for a binary tree node.
 * function TreeNode(val, left, right) {
 *     this.val = (val===undefined ? 0 : val)
 *     this.left = (left===undefined ? null : left)
 *     this.right = (right===undefined ? null : right)
 * }
 */

/**
 * 968. 监控二叉树
 *
 * 解题思路：
 * 使用“后序遍历 + 贪心”。
 *
 * 我们定义每个节点有 3 种状态：
 * 0 -> 当前节点没有被监控
 * 1 -> 当前节点上放了摄像头
 * 2 -> 当前节点已经被监控，但当前节点没有摄像头
 *
 * 状态转移规则：
 * 1. 如果某个孩子没有被监控（状态 0），那当前节点必须放摄像头（状态 1）
 * 2. 如果某个孩子有摄像头（状态 1），那当前节点已被监控（状态 2）
 * 3. 如果两个孩子都已被监控但都没有摄像头（状态 2），
 *    那当前节点此时没有被监控（状态 0）
 *
 * 注意：
 * 空节点返回状态 2（已被监控），因为空节点不需要摄像头。
 *
 * @param {TreeNode} root
 * @return {number}
 */
var minCameraCover = function(root) {
    // 用来记录最少摄像头数量
    let cameraCount = 0;

    /**
     * 后序遍历函数
     * 返回当前节点的状态：
     * 0 -> 未被监控
     * 1 -> 放了摄像头
     * 2 -> 已被监控
     *
     * @param {TreeNode} node
     * @return {number}
     */
    function dfs(node) {
        // 空节点：视为“已被监控”
        // 原因：空节点不需要摄像头，也不会影响父节点决策
        if (node === null) {
            return 2;
        }

        // 先递归处理左右子树（后序遍历）
        const leftState = dfs(node.left);
        const rightState = dfs(node.right);

        // 情况 1：
        // 只要左右孩子中有一个“未被监控”，
        // 那当前节点就必须放摄像头
        if (leftState === 0 || rightState === 0) {
            cameraCount++;
            return 1; // 当前节点放了摄像头
        }

        // 情况 2：
        // 如果左右孩子中有一个放了摄像头，
        // 那当前节点已经被监控到了
        if (leftState === 1 || rightState === 1) {
            return 2; // 当前节点已被监控，但没有摄像头
        }

        // 情况 3：
        // 左右孩子都“已被监控”，但都没有摄像头
        // 说明当前节点还没有被监控
        return 0;
    }

    // 获取根节点状态
    const rootState = dfs(root);

    // 如果根节点最后还没有被监控，
    // 由于根节点没有父节点，只能自己再放一个摄像头
    if (rootState === 0) {
        cameraCount++;
    }

    return cameraCount;
};
```

## 六、举个简单例子理解

比如一棵只有 3 个节点的链：

```text
    0
   /
  0
 /
0
```

从下往上看：

1. 最底下叶子节点左右为空，所以它返回 `0`（未被监控）
2. 中间节点发现左孩子是 `0`，所以它必须放摄像头，返回 `1`
3. 根节点发现左孩子是 `1`，说明根节点已被监控，返回 `2`

最终只需要 **1 个摄像头**。

## 八、这题的贪心本质

这题不是在“当前节点要不要装摄像头”上盲目决策，
而是遵循一个关键原则：

> **只有当某个孩子没被覆盖时，才在当前节点放摄像头。**

因为摄像头放在父节点，收益通常比放在叶子节点更大，能多覆盖一个父节点和更多分支。

# 先解释关键变量 / 状态分别表示什么

这段代码里，最关键的是下面这些变量和状态：

## 1）`cameraCount`

表示**当前已经放了多少个摄像头**。

* 初始值是 `0`
* 每当代码执行到：

```js
cameraCount++;
```

就说明：当前节点必须放一个摄像头，于是摄像头总数加 `1`

---

## 2）`dfs(node)`

这是一个递归函数。

它的作用不是直接返回“需要几个摄像头”，而是：

* 先递归处理 `node.left`
* 再递归处理 `node.right`
* 最后根据左右孩子的状态，返回**当前节点的状态**

也就是说，`dfs(node)` 返回的是 **状态值**，不是摄像头数量。

---

## 3）三个状态值的含义

### 状态 `0`

表示：

> 当前节点**没有被监控**

注意，这里不是说“这个节点没有摄像头”这么简单，而是更严重一点：

* 当前节点自己没有摄像头
* 当前节点也没有被孩子或父节点的摄像头覆盖到（至少在当前递归这一层看，它还没被覆盖）

---

### 状态 `1`

表示：

> 当前节点**放了摄像头**

只要当前节点返回 `1`，就说明代码在这个节点执行了：

```js
cameraCount++;
return 1;
```

---

### 状态 `2`

表示：

> 当前节点**已经被监控了，但当前节点没有摄像头**

也就是：

* 当前节点自己没放摄像头
* 但是它被它的孩子上的摄像头照到了，或者它作为空节点被视作已监控

---

## 4）`leftState` 和 `rightState`

这两个变量表示：

* `leftState = dfs(node.left)`：左子树处理完成后，左孩子的状态
* `rightState = dfs(node.right)`：右子树处理完成后，右孩子的状态

当前节点到底返回 `0 / 1 / 2`，完全由这两个值决定。

---

## 5）为什么空节点返回 `2`

代码里：

```js
if (node === null) {
    return 2;
}
```

意思是：

> 空节点直接视为“已被监控”

原因不是“空节点真的被照到了”，而是为了让父节点好判断，不必为不存在的孩子额外放摄像头。

你可以把它理解成：

* 空位置不用装摄像头
* 也不用担心它没被监控
* 所以把它当作“已处理、不会添麻烦”的状态 `2`

---

# 示例 2 整个过程结束后，为什么返回这个结果

最终返回 `2`，原因是：

1. 叶子节点 `E` 返回 `0`，表示自己未被监控
2. `D` 看到自己的右孩子 `E` 是 `0`，所以必须放摄像头，`cameraCount` 从 `0` 变成 `1`，`D` 返回 `1`
3. `C` 因为孩子 `D` 有摄像头，所以 `C` 返回 `2`
4. `B` 的左右孩子状态是 `2` 和 `2`，说明孩子都被处理好了，但没有摄像头照到 `B`，所以 `B` 返回 `0`
5. `A` 看到自己的左孩子 `B` 是 `0`，所以 `A` 必须放摄像头，`cameraCount` 从 `1` 变成 `2`
6. 根节点最终返回 `1`，不需要额外补摄像头

所以总共要放两个摄像头：

* 一个放在 `D`
* 一个放在 `A`

因此返回 `2`。

# 最后，用一句“人话”总结这段代码到底是怎么工作的

这段代码不是“看到一个节点就决定装不装”，而是：

* 先一路递归到最下面
* 先看孩子是什么状态
* 再让当前节点根据孩子状态做决定

它的核心因果链是：

* **孩子没被监控（0）** → **当前节点必须装摄像头（1）**
* **孩子有摄像头（1）** → **当前节点已经被监控（2）**
* **孩子都只是被监控但没摄像头（2）** → **当前节点反而没人管，所以是未监控（0）**

所以整个决策顺序非常依赖“后序遍历”——一定要先知道左右孩子状态，当前节点才能做出正确选择。
