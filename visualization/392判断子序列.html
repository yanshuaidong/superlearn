392. 判断子序列

这个页面不是只告诉你答案对不对，而是把这段双指针代码“拆开、放慢、可视化”。 你会看到每一步到底比较了什么、为什么有时只移动 j、为什么匹配成功时 i 才前进。

核心目标 判断字符串 s 能不能按顺序出现在字符串 t 中。
算法抓手 两个指针同时工作：i 看 s，j 扫 t。
最重要的规则 无论匹不匹配，j 都继续向后；只有匹配时 i 才前进。
模块一

题目与答案代码

先把题目和答案代码读顺。重点不是背代码，而是知道每一行在“控制什么行为”。

题目理解

给定两个字符串 s 和 t，判断 s 是否为 t 的子序列。
“子序列”强调的是：字符顺序不能乱，但中间可以跳过一些字符。 
/**
 * 判断字符串 s 是否为字符串 t 的子序列
 *
 * 双指针思路：
 * - i 指向 s 当前想匹配的字符
 * - j 指向 t 当前扫描到的字符
 *
 * 只要两个指针都还没越界，就继续比较：
 * 1. 如果 s[i] === t[j]
 *    说明在 t 中找到了 s 的当前字符
 *    这时 i++，去找 s 的下一个字符
 * 2. 不管有没有匹配成功
 *    j 都必须继续后移，因为 t 的扫描不能停在原地
 *
 * 最后如果 i === s.length
 * 说明 s 的每个字符都已经按顺序匹配完成
 */ 
var isSubsequence = function(s, t) {
    // i 用来遍历字符串 s
    let i = 0;

    // j 用来遍历字符串 t
    let j = 0;

    // 只要 s 还没匹配完，并且 t 还没扫描完，就继续
    while (i < s.length && j < t.length) {
        // 当前字符相同，说明找到了一个需要的字符
        if (s[i] === t[j]) {
            i++; // s 前进到下一个待匹配字符
        }

        // 无论匹配是否成功，t 都继续向后扫描
        j++;
    }

    // 如果 i 走到 s 末尾，表示 s 全部匹配完成
    return i === s.length;
};

读代码时要抓住的 4 个点

点 1：为什么用两个指针？
因为我们想一边记住 s 正在等哪个字符，一边不停扫描 t。
点 2：为什么 j 总要前进？
因为 t 是“被搜索的长串”，每个位置最多看一次，跳过就继续往后。
点 3：为什么匹配时只让 i 前进一次？
因为当前这一个字符只解决了 s 的一个需求，不能一下跳过多个字符。
点 4：循环结束后为什么看 i === s.length？
这表示 s 里每个字符都已经按顺序找到了，算法任务真正完成。
