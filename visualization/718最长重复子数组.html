718. 最长重复子数组

这不是只看结论的题解页，而是一份“跟着代码跑一遍”的教学页面。重点是看懂：为什么状态要定义成“以这两个位置结尾”、公式背后到底替代了哪些暴力枚举，以及代码里的每一步是怎样推动答案变大的。

题目内容

给两个整数数组 nums1 和 nums2，请找出它们共同拥有的、长度最长的“连续子数组”，并返回这个长度。

你要特别注意两个词：

1. 共同拥有：说明要同时出现在两个数组里。

2. 连续子数组：说明不能跳着选，一旦中间断掉，这一段就结束了。

例如：[3,2,1] 是 [1,2,3,2,1] 和 [3,2,1,4,7] 的共同连续子数组，长度是 3。

关键词：连续
关键词：两个数组同时出现
目标：最长长度

答案代码

/**
 * 718. 最长重复子数组
 * 动态规划解法
 *
 * @param {number[]} nums1
 * @param {number[]} nums2
 * @return {number}
 */
var findLength = function(nums1, nums2) {
    // 先拿到两个数组的长度，后面循环和建表都要用
    const m = nums1.length;
    const n = nums2.length;

    /**
     * dp[i][j] 表示：
     * 当我们强行要求公共子数组必须“刚好结尾”在
     * nums1[i - 1] 和 nums2[j - 1] 这两个位置时，
     * 这时能得到的最长公共连续子数组长度。
     *
     * 这样定义的关键好处：
     * 只有当前位置两个数相等时，前面的连续段才有资格继续往后接，
     * 所以它只会从左上角 dp[i - 1][j - 1] 延长过来。
     *
     * 多开一行一列空边界：
     * dp[0][*] 和 dp[*][0] 都表示“有一边是空数组”，结果自然是 0，
     * 这样写转移时就不用反复讨论 i = 0、j = 0 的特殊情况。
     */
    const dp = Array.from({ length: m + 1 }, () => Array(n + 1).fill(0));

    // 全局答案不一定落在最后一个格子，所以要边算边维护最大值
    let maxLen = 0;

    // 从 1 开始，是因为第 0 行和第 0 列专门留给“空边界”
    for (let i = 1; i <= m; i++) {
        for (let j = 1; j <= n; j++) {
            // 现在尝试把 nums1[i - 1] 和 nums2[j - 1] 对齐来看
            if (nums1[i - 1] === nums2[j - 1]) {
                // 这两个位置相等，说明：
                // 1. 至少能形成长度为 1 的公共连续段
                // 2. 如果左上角本来就有一段连续公共后缀，
                //    那么当前这两个相等元素可以把那一段整体再往后延长 1 位
                dp[i][j] = dp[i - 1][j - 1] + 1;

                // 当前格子可能刷新全局最优
                maxLen = Math.max(maxLen, dp[i][j]);
            } else {
                // 一旦这两个位置不相等，
                // 以它们作为共同结尾的连续子数组就不可能存在，
                // 因为“连续”被当前这一步直接打断了
                dp[i][j] = 0;
            }
        }
    }

    return maxLen;
};

看代码时先建立的整体印象

  • 这张 DP 表不是在存“前 i 个、前 j 个的答案”，而是在存“如果结尾固定在这两个位置，连续公共段能有多长”。
  • 为什么必须强调“结尾”：因为连续子数组只能整段接在前一对位置后面，所以来源只可能是左上角，而不是左边或上边。
  • 相等时加 1 的含义不是凭空背公式，而是“左上角已经枚举好的所有连续后缀，都可以整体加上当前这一位”。
  • 不相等时直接归 0，不是因为“不想继承”，而是因为“连续”这个条件让所有以当前这两个位置结尾的候选都失效了。
  • maxLen 单独维护，因为最长重复子数组可能出现在表中的任何一个格子，不一定在最后。
这一题最容易混淆的点： 它和最长公共子序列不一样。子序列可以跳着选，所以会看左、上、左上；这题要求连续，所以只看“左上角能不能接上当前这一位”。这一点会在下面的动画里被完整展开。
